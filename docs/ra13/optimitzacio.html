RA-14 - Reconeix les característiques de la generació de codi intermedi per a la seva execució a màquines virtuals

Optimització i Eficiència


El codi intermedi no només actua com a pont entre el codi font i la màquina virtual, sinó que permet aplicar tècniques d'optimització avançades per millorar el rendiment. Aquestes optimitzacions es realitzen en dues etapes clau:


Tècniques d'Optimització en Codi Intermedi

  • Reducció de Redundàncies:
    - Eliminació de codi mort (blocs inaccessibles).
    - Fusió de constants (ex: 3 + 5 → 8).
    - Propagació de constants a variables.

  • Inlining de Funcions:
    - Incrustació de funcions petites al codi que les crida (ex: mètodes getters en Java) per evitar càrrega de cridades.

  • Optimització de Bucles:
    - Loop unrolling: Desplegar iteracions per reduir condicionals.
    - Loop-invariant code motion: Moure càlculs invariables fora del bucle.

  • Anàlisi de Fluxe de Dades:
    - Identificar variables no utilitzades o assignacions redundants.

El Rol del Compilador JIT (Just-In-Time)

El compilador JIT és clau per aconseguir eficiència en temps d'execució:

  1. Fase d'Interpretació: Execució inicial del bytecode sense optimitzacions.

  2. Detecció de "Hot Spots": Identifica les parts del codi més executades (ex: bucles intensius).

  3. Compilació a Codi Màquina: Converteix el codi intermedi a instruccions natives de la CPU, aplicant:

    • Especialització de tipus (ex: optimitzar per a enters si una variable sempre és int).
    • Inline Caching per a mètodes polimòrfics (comú en Java).

  4. Reoptimització Dinàmica: Adapta el codi en temps real segons el comportament de l'aplicació.

Exemple Pràctic: JVM HotSpot

La màquina virtual de Java (JVM) utilitza el compilador HotSpot JIT per aplicar optimitzacions com:

  • Eliminació de bloquejos redundants en fils d'execució.
  • Vectorització d'instruccions per a processament paral·lel.
  • Optimització de crides a mètodes virtuals.
